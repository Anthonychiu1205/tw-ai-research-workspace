# Demo Walkthrough

Use this flow when presenting the workspace to external users.

1. Open `/workspace`.
2. Confirm current mode (`mock` by default) and backend status.
3. Start `2330` demo from onboarding.
4. Inspect created research artifact and signal summary.
5. Open evidence references and selected timeline items.
6. Open planner trace and reflection notes.
7. Generate a research report and review section-level evidence.
8. Compare watchlist strategy outputs.
9. Run portfolio review (target allocation simulation).
10. Run backtest v2 summary (hypothetical, non-execution).
11. Export a workspace share bundle.
12. Close with limitations and safety statement.

## Safety language for demos

- synthetic research outputs
- hypothetical backtest metrics
- non-advice disclaimer
- no broker integration
- no trade/order execution
