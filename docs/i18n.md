# i18n

## Supported locales

- `zh-TW` (default)
- `en-US`

## Architecture

- dictionaries:
  - `lib/i18n/dictionaries/zh-tw.ts`
  - `lib/i18n/dictionaries/en-us.ts`
- provider and lookup:
  - `lib/i18n/i18n-context.tsx`
- hook:
  - `lib/i18n/use-i18n.ts`
- persistence:
  - `lib/i18n/i18n-store.ts`

## How keys work

- Use path lookup style: `t("nav.workspace")`
- Missing key fallback order:
  1. active locale
  2. `en-US`
  3. raw key string

## Persistence and default

- default locale: `zh-TW`
- user choice saved in localStorage
- SSR-safe locale access (no direct window usage on server)

## What not to translate

- enum values
- JSON keys
- API route paths
- function names
- schema fields
- model ids
- provider ids

## Safety wording policy

- localized copy must keep non-advice wording
- localized copy must not introduce buy/sell/trading execution claims
- mock/synthetic boundary language must remain explicit
