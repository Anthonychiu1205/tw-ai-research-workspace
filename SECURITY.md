# Security Policy

## Reporting

Please do not disclose vulnerabilities publicly before coordinated review.
Contact maintainers through a private channel.

## Security expectations

- Never commit real API keys or secrets.
- Keep `.env.example` placeholder-only.
- Keep mock-first default behavior.
- Keep no-trading / no-broker boundaries.

## Dependency and supply-chain hygiene

When reporting dependency concerns, include:
- package name and version
- vulnerability reference (if available)
- reproduction details

## Supported versions

- `v1.0.x`
- `main` (latest stable branch)
